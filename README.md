# solarscraper-homeass

Scrapes solar inverter data from a cloud API and pushes it to Home Assistant via the REST API. Runs on a 5-minute polling interval.

## Sensors

The following Home Assistant entities are updated:

| Entity | Unit | Description |
|---|---|---|
| `sensor.solar_soc` | % | Battery state of charge |
| `sensor.solar_pv_power` | W | Current PV generation |
| `sensor.solar_load_power` | W | Current household load |
| `sensor.solar_grid_power_from` | W | Power drawn from grid |
| `sensor.solar_grid_power_to` | W | Power exported to grid |
| `sensor.solar_batt_power_from` | W | Power discharged from battery |
| `sensor.solar_batt_power_to` | W | Power charging to battery |
| `sensor.solar_gen_power` | W | Generator power |
| `sensor.network_outage` | bool | True if data is stale (>10 min old) |
| `sensor.solar_consumed_to_load` | kWh | Daily energy consumed by load |
| `sensor.solar_consumed_to_grid` | kWh | Daily energy exported to grid |
| `sensor.solar_consumed_to_battery` | kWh | Daily energy charged to battery |
| `sensor.solar_consumed_from_pv` | kWh | Daily energy generated by PV |

## Setup

```bash
npm install
cp .env.example .env   # then fill in your values
```

### Environment variables

| Variable | Description | Required |
|---|---|---|
| `SOLARK_USERNAME` | Cloud API username | Yes |
| `SOLARK_PASSWORD` | Cloud API password | Yes |
| `HA_URL` | Home Assistant base URL (e.g. `http://192.168.1.10:8123`) | Yes |
| `HA_TOKEN` | Home Assistant long-lived access token | Yes |

## Usage

```bash
node solarpower.js
```

The app fetches data immediately on startup, then every 5 minutes.

## Running as a service

```ini
# ~/.config/systemd/user/solarscraper.service
[Unit]
Description=Solar Scraper to Home Assistant
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/path/to/solarscraper
ExecStart=/usr/bin/node solarpower.js
Restart=always
RestartSec=10

[Install]
WantedBy=default.target
```

```bash
systemctl --user enable --now solarscraper
loginctl enable-linger $USER
```

## How it works

1. Authenticates against the cloud API using OAuth2 password grant
2. Fetches three endpoints: energy flow, realtime usage, and generation/consumption
3. Merges the responses and detects network outages (data older than 10 minutes)
4. POSTs each sensor value to Home Assistant's REST API
5. Automatically refreshes the access token on 401 responses
